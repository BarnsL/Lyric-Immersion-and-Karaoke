### 4. Concert chapter setlist recognition (v1.1.62)

Substantial pipeline addition for full concert videos.

- **Description-embedded setlist parser** at `yt_description.py:parse_setlist_timestamps` — pulls `MM:SS Title` / `HH:MM:SS Title` lists from video descriptions with monotonic + header + length guards. Wired into `_load_concert_setlist` at `main.py:5309` when native chapters are absent.
- **Chapter-intro hold** — same freeze-until-vocals mechanism as MV mode, engaged when a chapter fires without an offline vocal-onset. State fields `_chapter_intro_hold`, `_chapter_intro_pos0`, `_chapter_intro_t0` at `main.py:3321`.
- **Stale-lyrics clear** — on a chapter switch with a fetch in flight, `self.lines` / `self.idx` / `self._kara` are cleared at `main.py:5526` so `_on_vocal_onset` can't compute a wrong offset from the previous chapter's timeline.
- **Sentence-length gate skip** for description-parsed setlists — description parser has its own 80-char guard, so the 40-char "documentary section" heuristic would over-reject legitimate concert setlists with credit-annotated titles.
- **Auto-persist restore on concert plan install** — `_apply_concert_plan` re-ticks `_concert_setlist_tick` for the current position when the offline plan lands mid-playback, so a chapter that engaged the crude hold picks up the newly-arrived vocal onset.

### 5. Auto game-focus mode (v1.1.62)

New feature: detect a D3D exclusive-fullscreen game in focus, auto-swap to Gaming preset, restore prior preset on game exit.

- **Detection** `_is_game_in_focus` at `main.py:10695` — three layers:
  1. `gpu_setup.game_active()` (SHQueryUserNotificationState == 3 == D3D fullscreen). Same API Windows uses for Focus Assist / Game Mode / Auto-HDR — deliberately excludes browsers, video players, our overlay.
  2. Foreground window PID ≠ our own PID.
  3. Foreground window is fullscreen on its monitor (multi-monitor guard: game on M1, browser on M2 → don't swap).
- **State machine** in `_check_game_focus` at `main.py:10820`:
  - 1.5 s arm delay before swap.
  - 4 s release delay before restore (absorbs alt-tab flap).
  - Tracks previous preset ('karaoke' / 'subtitles' / None for custom state).
  - Auto apply/restore call `apply_preset(name, _from_auto=True)` — critical: **auto swaps skip `_persist()`** so the gaming preset never overwrites the user's original preset on disk. If app is closed mid-game, restart brings back the pre-game preset.
  - Manual `apply_preset(name)` while auto-swapped cancels tracking so the user's choice sticks.
- **Tray toggle** — new "🎮 Auto game-focus" checkbox in the tray menu, persisted as `auto_game_mode`.

### 6. Gaming preset overhaul + tight sync (v1.1.62)

Both fixed — `_from_auto=True` skips persist entirely, fullscreen guard rejects windowed-foreground scenarios.

## Known-open / pending

1. **Candidate pool is EXTRACTED but not yet routed to a matching stage.** The list is stored on `self._concert_candidates` and logged, but no downstream code consumes it yet. Next step: fetch each candidate's lyrics on load (background thread), then teach the by-ear identification path (`_decide_by_ear` / `_live_resync_loop`) to search only within this pool rather than doing blind global Shazam. This is the last mile for making lyrics actually appear on livestream concerts with no chapter metadata. User was informed and knows this is the next explicit ask.
2. **Inno Setup (`iscc`) not on PATH** so `Setup.exe` isn't built by `build.bat`. Portable deploy works fine via robocopy mirror. If a release is cut, need to add `iscc` to PATH or invoke the installer step separately.
3. **`Get-FileHash` missing in the ancient PowerShell** invoked by `build.bat`'s `powershell -NoProfile -Command` — the zip is written but the `.sha256` companion is empty. Fine for local deploy; matters only for public GitHub Releases (in-app updater needs the hash). Fix: change `powershell` to `pwsh` in build.bat or add a fallback hash step.
4. **VTuber borderless-windowed games** don't trip `SHQueryUserNotificationState` (returns 3 only for exclusive fullscreen). Some modern games (Valorant, Overwatch, most Unreal 5 titles) run borderless-windowed by default. Auto game-focus won't fire for those. Not a bug — Windows itself uses the same signal, so we're at parity with Windows Focus Assist / Game Mode.
